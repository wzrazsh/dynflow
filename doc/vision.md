# Dynamic Workflow Engine - 产品愿景

## 核心使命

提供一个轻量、零依赖的 TypeScript SDK，让开发者能够用代码编排多代理 LLM 工作流，实现并行执行、状态恢复和智能缓存。

## 我们做什么

### 核心价值
- **编排多代理 LLM 任务**：定义阶段、任务、并发控制，让多个 AI 代理协同工作
- **零运行时依赖**：只使用 Node.js 18+ 内置模块，确保轻量级和可移植性
- **开发者友好**：提供 Config API 和 Builder API 两种使用方式
- **生产就绪**：内置重试、超时、缓存、会话恢复等生产级特性

### 目标用户
- 需要编排复杂 AI 工作流的开发者
- 构建多代理系统的团队
- 需要可靠、可恢复 LLM 调用的应用

## 我们不做什么

### 明确边界
1. **不提供 LLM 服务**：我们是 SDK，不提供模型 API
2. **不提供 UI 界面**：纯代码库，无可视化编辑器
3. **不管理模型训练**：专注于推理和编排，不涉及模型训练
4. **不提供持久化存储**：使用内存缓存，持久化是可选功能
5. **不绑定特定 LLM 提供商**：支持任何 OpenAI 兼容 API

### 技术边界
- **Node.js 仅限**：不支持浏览器环境
- **TypeScript 优先**：虽然可以编译为 JS，但类型安全是核心
- **异步优先**：所有操作都是异步的，不支持同步调用

## 核心原则

1. **简单性**：API 设计简洁直观，避免过度抽象
2. **可靠性**：错误处理完善，确保工作流可恢复
3. **性能**：并行执行，智能缓存，避免不必要的开销
4. **可扩展性**：通过插件和配置支持各种使用场景
5. **开发者体验**：优秀的文档、类型提示和错误信息
